## Step 1 — Understand the folder structure first

Before touching code, understand WHY this structure exists:

```
project/
├── controllers/     # handles HTTP requests/responses only
├── services/        # business logic lives here
├── repositories/    # all database queries live here
├── models/          # database table definitions
├── dto/             # what data looks like coming IN (request shapes)
├── validators/      # extra validation rules
├── middlewares/     # logging, auth, tracing
├── config/          # env vars, settings
└── main.py          # app entry point
```

The rule is: each layer only talks to the layer below it. Controller → Service → Repository → Database. Never skip layers.
